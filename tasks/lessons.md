# 经验记录

- 2026-04-15：`tasks` 目录中的任务说明与跟踪文档统一使用中文，避免中英文混用影响项目内协作。
- 2026-04-15：新增安全链路这类跨模块能力时，要同步补上“为什么这样做”的维护型注释，尤其是签名协议、拦截器顺序和兼容回退逻辑，避免后续维护者只能靠读调用链猜设计意图。
- 2026-04-16：像 `JwtUtils` 这种安全基础设施类，注释要明确“只负责哪一层校验、哪些事情不在这里保证”，否则后续维护时很容易把 JWT 自身合法性和 Redis 登录态有效性混为一谈。
- 2026-04-16：跨服务通用能力不要先落在单个业务模块里凑合使用。像内部 Feign 签名拦截器这类公共能力，应优先放到公共安全模块并用条件装配控制生效范围。
- 2026-04-16：凡是把 path 纳入签名的请求，不要在 Feign `RequestTemplate` 阶段计算签名；要尽量绑定最终 `Request`，否则类级 path、target URL 等后续拼接信息可能丢失。
