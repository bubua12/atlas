# 经验记录

- 2026-04-15：`tasks` 目录中的任务说明与跟踪文档统一使用中文，避免中英文混用影响项目内协作。
- 2026-04-15：新增安全链路这类跨模块能力时，要同步补上“为什么这样做”的维护型注释，尤其是签名协议、拦截器顺序和兼容回退逻辑，避免后续维护者只能靠读调用链猜设计意图。
- 2026-04-16：像 `JwtUtils` 这种安全基础设施类，注释要明确“只负责哪一层校验、哪些事情不在这里保证”，否则后续维护时很容易把 JWT 自身合法性和 Redis 登录态有效性混为一谈。
- 2026-04-16：跨服务通用能力不要先落在单个业务模块里凑合使用。像内部 Feign 签名拦截器这类公共能力，应优先放到公共安全模块并用条件装配控制生效范围。
- 2026-04-16：凡是把 path 纳入签名的请求，不要在 Feign `RequestTemplate` 阶段计算签名；要尽量绑定最终 `Request`，否则类级 path、target URL 等后续拼接信息可能丢失。
- 2026-04-16：MyBatis-Plus 从 `StdOutImpl` 切到 `Slf4jImpl` 后，SQL 是否可见取决于 `logging.level` 是否对 mapper 命名空间开到 `DEBUG/TRACE`；给方案时必须把日志实现和 mapper 包级日志配置一起说明。
- 2026-04-16：命令行脚本如果读取 `ATLAS_NACOS_HOST` 这类环境变量，不能假设用户一定会带端口；要自动兼容 `host` 和 `host:port` 两种写法，并在执行前打印最终 endpoint 便于排障。
- 2026-04-16：给 Nacos 准备导入模板时，不要默认沿用本地 `application.yml` 的环境变量占位符；如果用户希望配置中心直接托管真实值，模板就应该提供可直接导入的字面值。
- 2026-04-16：向 Nacos 发布配置时，除了 `dataId/group/content` 以外，还要显式传 `type`；否则控制台可能把 YAML 展示成普通文本格式。
- 2026-04-16：当用户明确希望“能迁到 Nacos 的都迁进去”时，不要只迁数据库这类基础配置；端口、路由、签名密钥、服务私有开关等可远程化项也应一起迁走，本地配置尽量收敛到连接配置中心所需的最小集合。
